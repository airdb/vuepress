### 2.2 使用说明

工作计划分为3段，第1段表示工作进度，第2段表示分类及优先级，第三段表示具体的工作内容。

a. 进度标识

- [ ] : 设定工作计划
- [x] : 已完成
- [n] : 推迟下周开展
- [c] : 下周继续
- <del> [d] : 删除该项工作计划 </del>

b. 分类标识

- :new:  : 新功能、新模块
- :bug:  : 设计缺陷、漏洞或者 bug, 可以正常迭代
- :rotating_light: : 一级紧急, 生产 bug 且有一定用户影响
- :red_circle: : 重要且紧急
- :warning:  : 重要不紧急
- :golf: : 后续要做，先设定为计划

c. 工作内容

单条工作内容尽量不超过20个汉字，目标必须可以在9小时以内完成，且能够进行验证、验收。
